Ornge Project Documentation

Project Overview:

We are developing our web application using JavaScript, and Python programming language including (React.js and Django). Our core application development is in progress, and we are now focused on adding new features and accommodating change requests. Ornge is primarily designed for the 3D printing of braces and aligners. We have our web application that is used to manipulate the 3D scans of jaws, handle all braces, aligners, attachments and many more things that are needed by Orthodontists for treatment planning. There is a wide range of features that Orthodontists can use. We manage development using Jira and our code is managed by Github

Project Details:

Different Components of the Project:

  • Frontend codebase for the Web portal (Javascript (React, Next))
  • Backend codebase for the public portal (Python (Django))

Project Environments:

For now, as our project is in the initial phase, we have a staging environment only for both the frontend and backend applications.

New Developer Onboarding:

Tools Needed:

  • Visual Studio
  • Node.js (latest stable version preferred)
  • Python (latest)
  • Global Protect (optional)

Development Machine Setup:

Ensure the tools listed above are installed. Set up Git using the following commands:

git config --global user.name "Your Fullname"
git config --global user.email "Your Email address"

For GitHub access, refer to GitHub SSH setup instructions. Clone the application:

git clone https://github.com/orngeteam/ornge-frontend.git
git fetch && git checkout development

Create a .env file and populate it with the required configurations.

Building the Application:

Run the following commands to build the app:

yarn install
yarn start

Access the application at http://localhost:3000.

Accounts:

For new team members, access is needed for:

  • Jira: Task management
  • GitHub: Repository access
  • AWS: Optional for service manipulation
  • Ornge: Account creation on qa.ornge.com (e.g., test@ornge.com with password dev@12345678)

Development Process:

The application uses a modified feature model for development. Developers work in feature branches, with the development branch as the base. For hotfixes, a hotfix branch is created from the master branch. Below is a sample folder structure for development:

|-- arbisoft
    |-- ornge-team
        |-- ornge-frontend
        |-- ornge-backend

Commit Template and Branches:

Set up the local Git commit template with the following format:

:gitmoji: <title>

<empty-line>

Ticket: https://ornge.atlassian.net/browse/OR-000

Example commit:

:sparkles: OR-123 implements new feature

Branch names follow this format:

<dev-name>/ticket-no-one-liner

Example:

haniya/OR-111-fixes-something

Code Review Process:

At Ornge, code reviews ensure code quality. Reviewers focus on functionality, complexity, tests, naming conventions, comments, and documentation. Pair programming and in-person reviews are encouraged.

Release and Deployment:

Ornge follows a straightforward release and deployment process. Releases are documented on GitHub, and deployments occur as needed, especially during maintenance phases.
